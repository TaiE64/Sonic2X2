# 跨机器人策略迁移 Checklist

从 SONIC(G1) → AgiBot X2 复现中踩出来的 16+ 个 bug 提炼而成。
每一条都对应一次真实的返工。**下次做任何 A→B 机器人迁移,逐条打勾。**

判据: 每一项都必须有**独立来源的数值证据**才算通过。"看代码觉得对"不算,"自测通过"不算(自测可能用同一个错误公式互证)。

---

## 一、索引与映射 (最高危区,占全部严重 bug 的一半)

- [ ] **每张置换/索引数组,对每个槽位验证 `resolve(name) == name`** — 不许抽查几个末端就下结论
  - 踩过: body 映射表从 G1 抄来漏了脚趾,25/32 槽位错位,贯穿 13 个版本;抽查腕/踝/头恰好绕过了坏表
- [ ] **验证用的真值必须来自独立来源** (真实 env 探针 dump,不是自己按同一公式构造的假环境)
  - 踩过: 自测用同一个反向公式构造假 env,"自洽错误互证",测试通过但线接错了 28/31
- [ ] **写明索引约定式**并实测哪个方向成立 (`q_mj = q_il[IL2MJ]` 还是 `q_il = q_mj[IL2MJ]`)
  - 踩过: 审计脚本自己把 IL2MJ 当 MJ2IL 用,报出"13/31 缩放不一致"的假警报(靠"腕配到髋"这种荒谬结果才察觉)
- [ ] **FK 骨架的 body 数 ≠ 仿真器的 body 数** — 装饰性节点(脚趾/手指)会让两边长度不等,长度不等本身就是红灯
  - 踩过: FK 34 body(含 toe) vs IsaacLab 32 body
- [ ] **变量名可能撒谎** — `XXX_JOINTS` 里装的可能是 body 名
- [ ] 全树 grep 索引数组名,**确认没有重复定义/本地拷贝**,消费方不会 import 到旧版本

## 二、语义配对 (按名字配对是陷阱)

- [ ] **关节链的语义位置 > 关节名字** — 两个机器人的同名关节可能在链上位置不同
  - 踩过: G1 腕链 roll→pitch→yaw,X2 是 yaw→pitch→roll。按名配对 = 把前臂旋转接到末端小关节,动作缩放错 5.65 倍
- [ ] **对每对匹配关节做 FK 方向审计**: 各给 +δ,比较子链的世界系旋转轴 dot
  - 踩过: 一对腕关节 dot = -1.0(方向完全相反),零位校准根本发现不了
- [ ] **看几何,别看名字**: 关节的偏移量和转轴决定它是什么
  - 踩过: X2 的 "wrist_yaw" 其实是前臂旋转(0.121m 连杆绕自身长轴),真正的腕是 pitch+roll 共点 2-DOF
- [ ] 语义映射表的**键类型要和消费方一致** (字面名 vs 正则模式)
  - 踩过: 交换表用字面关节名,但被交换的 dict 键是正则模式 → 交换永不命中,静默退化成按名配对

## 三、观测契约

- [ ] **逐观测项确认是相对量还是绝对量** (`joint_pos_rel = q - q_default` vs 绝对 `joint_pos`)
  - 踩过: 按绝对量处理相对量,肘部本体感受每帧偏 1.2 rad(69°)
- [ ] **reshape 布局绝不能猜**: `[pos块|vel块]` 还是逐帧 `[pos|vel]` 交错?对真值逐位验证
  - 踩过: 猜成交错,未来帧参考全乱(err ~10),策略拿垃圾目标
- [ ] **未来帧的时间步长/skip 要对真值验证** (frame_skip=1 还是 5?)
- [ ] 每一块 obs 都要对**关掉噪声**的真值逐位对齐 (`enable_corruption=False`)
- [ ] 别信 docstring
  - 踩过: `command_multi_future` 文档说是 body positions,实为 joint pos+vel

## 四、动作空间

- [ ] **动作缩放要按语义配对的源关节值设置**,而不是自己按 PD 推导
- [ ] 逐关节验证: 每个目标关节的 scale == 其源语义伙伴的 scale (全部,不是抽查)
- [ ] 确认动作路径是**纯线性映射**(无零位偏置) — 偏置只属于位置量
- [ ] 目标机器人多出的自由度(如头)确认被置零且不在控制回路

## 五、奖励 / 终止 / DR (配置树全量语义审计)

- [ ] **grep 生效 config 里每一个 body/joint 名**,逐个判断在新机器人上语义是否成立 — 一次做完,别修一个算一个
  - 踩过: 腕链语义漂移导致三处配置错(接触白名单/终止判据/DR质量) → "机器人学不会爬"
- [ ] **读消费函数的实现**,别只看配置 — 有些条目会被静默丢弃
  - 踩过: anti_shake 里的 head_pitch_link 不在 14-body 追踪列表 → 被过滤器静默丢弃,是死条目
- [ ] 终止阈值对照论文原文,别沿用源仓库的训练组
  - 踩过: 训练用了比论文严 2-4 倍的终止判据,压制长时程学习
- [ ] 接触惩罚白名单: 确认新机器人的**手掌/膝盖着地**不会被误罚(否则爬行/起身被结构性禁止)

## 六、软限位 / 惩罚带 (形态学差异的隐形杀手)

- [ ] **检查软限位公式在新机器人上的后果**: "中点±0.9×半程"在**行程窄且不对称**的关节上会把自然姿态划进惩罚区
  - 踩过: X2 肩 roll 行程 [-0.061, +2.993],软限下界 +0.092 → 手臂自然垂下就在挨罚(~100% 帧)
- [ ] 全数据集扫描: 统计**参考动作落在惩罚带内的帧占比** — 超过几个百分点就要警惕
  - 踩过: 腰 pitch 96% 的帧在惩罚带 → 机器人不敢前倾 → 全程后仰
- [ ] 检查**重定向输出是否饱和在硬限上** — 如果是,任何软限边距都会与跟踪目标冲突
- [ ] 惩罚权重 vs 跟踪收益的**量级对比**: 跟一步参考亏 -0.47、赚 +0.006 → 策略会理性地放弃跟踪
- [ ] ⚠️ 关闭软限位是仿真手段。**真机部署前必须恢复限位保护**(否则堵转烧电机)

## 七、数据管线

- [ ] **仿真地形 vs 参考动作的地形假设要匹配** — 平地仿真里放上下楼梯数据 = 物理不可能任务
  - 踩过: 250 条楼梯片段吃掉 70% 的失败采样预算,把课程绑架了
- [ ] **道具依赖动作**(坐椅子/搬箱子/开门)在无道具仿真里同样不可能 — 用 root 高度/接触签名筛出来
- [ ] **QC 阈值要对齐硬件真值** (速度上限用 URDF 的 velocity limit,不要拍脑袋)
  - 踩过: QC 用 15 rad/s,硬件只有 11.9 → 放进 528 条永远跟不住的片段
- [ ] **QC 失败 ≠ 数据不能用** — 先问"是数据烂还是当时管线不够好"
  - 踩过: 7887 条被毙,用升级后的管线(防碰撞 IK + 限速器 + 放宽时长)救回 5,913 条(75%)
- [ ] 四元数格式 (wxyz vs xyzw) 用**重力方向判据**验证,别信注释
- [ ] 数据的 fps 与 motion_lib 的 target_fps 不一致时,确认重采样正确且四元数重归一化

## 八、训练与判读 (最容易自欺的部分)

- [ ] **rollout 曲线不可信** — 自适应采样(PMCP)让批次越来越难,reward/error 变差可能只是"考卷变难"
  - 踩过 4 次,每次都差点误判为训练退化
- [ ] **唯一可信的健康信号 = 固定考卷**(一组恒定的保留片段,定期 merge 权重跑评估)
- [ ] **过训练会发生**: 内部 failure_rate 继续降,但独立考卷全面退步
  - 踩过: iter 12k → 21.8k,六项指标全退,跳跃腾空腰斩,而内部指标"更好看"
- [ ] LoRA-only 训练时**冻结 std** — 否则熵奖励会把"涨 std"当唯一捷径,导致死亡螺旋
  - 踩过: entropy 单调升 → 噪声↑ → 奖励崩塌 15 倍
- [ ] 检查 PPO 内脏 (KL / LR):LR 被压在下限说明 KL 失真,要查 rollout↔update 的 logprob 一致性
- [ ] 每次改数据集后,checkpoint 里的课程统计会失配 — 确认加载器的行为(重置 or 报错)

## 九、验证纪律 (贯穿以上所有)

- [ ] **用户/同事看到的现象要当真**。"我数值验证过了"和肉眼观察冲突时,**先怀疑验证的覆盖面**
  - 踩过多次: "参考骨架没问题"(其实错乱)、"腕在动"(其实末端腕没跟)、"是物理 gap"(其实还有 3 个真 bug)
- [ ] 荒谬的结果是礼物 — "腕配到髋"这种不合理配对,是发现自己审计脚本有 bug 的唯一线索
- [ ] 找不到根因时,**数学上免疫掉它**也是合法工程解(如 PPO 的 logprob 基线重算 pre-pass)
- [ ] 诊断探针**门控保留在代码里**,别用完就删 — 否则测量不可复现、下次还要重写
- [ ] 破坏性操作(删数据/改配置)前**多等一拍**,给用户改主意的时间窗
